# ブログSNS投稿生成（X & Threads）

## Description
`09_Blog/記事` フォルダにある WordPress ブログ記事ドラフトから、X.com（旧Twitter）と Threads 向けの宣伝投稿文を複数パターン生成し、同じフォルダ内に SNS 投稿案ファイルを出力します。

## 使用するフォルダ
- 記事フォルダ: `09_Blog/記事`

## Prompt
指定されたブログ記事をもとに、X.com と Threads で記事を紹介する日本語の投稿文を、それぞれ複数パターン作成してください。

## 引数
- 記事ファイル名（必須）
  - 例: `09_Blog/記事/20251123_ai-tools-comparison.md`
- 記事URL（任意）
  - 未指定の場合は `[記事URL]` というプレースホルダ文字列をそのまま出力する
- メインキーワード or ハッシュタグ候補（任意）
  - 例: `#AIツール`, `#WordPressブログ` など

## 実行内容
1. 記事本文の読み込み
   - 引数で渡された記事ファイルを `read_file` で全文読み込む
   - タイトル、リード文、見出し、まとめ部分から「一番おいしいポイント」を抽出

2. 想定読者とベネフィットの把握
   - 記事が誰に向けて書かれているか
   - 読んだあとにどんな変化・メリットがあるか

3. X.com 向け投稿案の作成
   - 2〜3 パターン程度
   - 1投稿あたり 140〜230 文字程度を目安
   - 冒頭 1 行で「フック（興味を引く一言）」を置く
   - 本文で読者の悩み → 記事の解決策・ベネフィット を簡潔に伝える
   - 最後に記事URL とハッシュタグを付与

4. Threads 向け投稿案の作成
   - 2〜3 パターン程度
   - X よりやや長めでもよい（改行を多めにして読みやすく）
   - ストーリー寄り・体験談寄りの切り口も含める
   - 最後に記事URL とハッシュタグを付与

5. ハッシュタグの設計
   - 記事テーマ、ジャンル（例: #AI, #ブログ, #副業, #WordPress など）
   - サイト・ブランド名があればそれも候補に含める
   - 1投稿あたり 3〜5 個程度を目安

6. 出力ファイルの作成
   - 元記事ファイル名を `basename.md` とする
   - SNS 投稿案ファイル名: `basename_SNS.md`
     - 例: 記事 `20251123_ai-tools-comparison.md` → `20251123_ai-tools-comparison_SNS.md`
   - 出力先: `09_Blog/記事/basename_SNS.md`
   - すでに同名ファイルが存在する場合は末尾に連番を付与（`_SNS-2.md` など）

## 出力形式

```markdown
## X.com向け投稿案

### パターン1
[本文（140〜230文字程度）]

[記事URL]
#ハッシュタグ1 #ハッシュタグ2 #ハッシュタグ3

### パターン2
[別の切り口の本文]

[記事URL]
#ハッシュタグ1 #ハッシュタグ2 #ハッシュタグ3

### パターン3（任意）
[さらに別の切り口の本文]

[記事URL]
#ハッシュタグ1 #ハッシュタグ2 #ハッシュタグ3

---

## Threads向け投稿案

### パターン1
[本文（少し長めでもOK、改行多めで読みやすく）]

[記事URL]
#ハッシュタグ1 #ハッシュタグ2 #ハッシュタグ3

### パターン2
[ストーリー寄り・体験談寄りの本文]

[記事URL]
#ハッシュタグ1 #ハッシュタグ2 #ハッシュタグ3

### パターン3（任意）
[別視点の本文]

[記事URL]
#ハッシュタグ1 #ハッシュタグ2 #ハッシュタグ3
```

## 手順（エージェント向け）
1. 記事ファイルを `read_file` で全文読み込み
2. 記事の要点・ベネフィット・ターゲット読者を抽出
3. X 向けに 2〜3 パターンの短文投稿を設計
4. Threads 向けに 2〜3 パターンのやや長めの投稿を設計
5. 記事URL 引数があればそれを挿入、なければ `[記事URL]` のまま出力
6. ハッシュタグを 3〜5 個程度選定し、各パターンに付与
7. 元記事ファイル名から `_SNS.md` を付けたファイル名を作成し、`09_Blog/記事` フォルダに書き出す

## 注意事項
- すべて日本語で作成すること
- 同じ文章のコピペにならないよう、パターンごとに切り口・トーンを変える
- X では短くキャッチーに、Threads では少し踏み込んだ内容・ストーリー性を意識する
- 絵文字使用の有無は記事トーンに合わせて柔軟に（ビジネス寄りなら控えめを推奨）
