# ブログSEOメタ生成（SEO SIMPLE PACK用）

## Description
`09_Blog/記事` フォルダにある WordPress ブログ記事ドラフトから、SEO SIMPLE PACK 用のタイトル（title）、ディスクリプション（description）、タグ（tags）を生成し、同じフォルダ内に SEO 設定案ファイルを出力します。

## 使用するフォルダ
- 記事フォルダ: `09_Blog/記事`

## Prompt
指定されたブログ記事本文をもとに、WordPress プラグイン **SEO SIMPLE PACK** で設定する SEO メタ情報（タイトル・ディスクリプション・タグ）を日本語で生成してください。

## 引数
- 記事ファイル名（省略時は現在開いているファイル）
  - 例: `09_Blog/記事/20251123_ai-tools-comparison.md`
- メインキーワード（任意、1語）
  - 未指定の場合は記事から自動抽出

## 実行内容
1. 対象記事ファイルを決定
   - 引数が指定されていればそのパスを使用
   - 指定がない場合は現在開いているファイルを使用
   - いずれの場合も `09_Blog/記事` 配下のファイルであることを確認

2. 記事本文の読み込み
   - `read_file` で記事ファイルを全文読み込む
   - タイトル、リード文、見出し構成、まとめ部分を中心に分析

3. メインキーワードの決定
   - 引数にメインキーワードがあればそれを優先
   - なければ本文からもっとも重要と思われる検索キーワードを 1 つ抽出

4. SEO SIMPLE PACK 用メタ情報の生成
   - タイトル（title）
     - 32〜40 文字程度の日本語タイトル
     - メインキーワードをできるだけ左側に含める
     - 検索ユーザーのベネフィットが伝わる形にする
   - ディスクリプション（description）
     - 全角 80〜120 文字程度
     - 冒頭でメインキーワードに近い表現を含める
     - 読者の悩み → 記事で得られる解決・メリット を自然な文章で説明
     - クリックしたくなる一文で締める
   - タグ（tags）
     - テーマ、カテゴリ、関連技術名などを 3〜7 個程度列挙
     - 日本語ベースで、ユーザーが検索しそうな語を優先

5. 出力ファイルの作成
   - 元記事ファイル名を `basename.md` とする
   - SEO 設定案ファイル名: `basename_SEO.md`
     - 例: 記事 `20251123_ai-tools-comparison.md` → `20251123_ai-tools-comparison_SEO.md`
   - 出力先: `09_Blog/記事/basename_SEO.md`
   - すでに同名ファイルが存在する場合は末尾に連番を付与（`_SEO-2.md` など）

## 出力形式

```markdown
## SEO SIMPLE PACK設定案

### タイトル（title）
[32〜40文字程度の日本語タイトル]

### ディスクリプション（description）
[全角80〜120文字程度の説明文]

### タグ（tags）
- [キーワード1]
- [キーワード2]
- [キーワード3]
- [必要に応じて追加]
```

## 手順（エージェント向け）
1. 記事ファイルパスを決定し、`read_file` で全文を取得
2. 本文から想定読者・テーマ・重要トピックを抽出
3. メインキーワードを決定（引数優先）
4. 上記ルールに沿ってタイトル・ディスクリプション・タグ案を生成
5. 元記事ファイル名から `_SEO.md` を付けたファイル名を作成
6. `09_Blog/記事` フォルダに SEO 設定案ファイルを書き出す

## 注意事項
- タイトル・ディスクリプションは日本語で自然な文章にすること
- 不自然なキーワード羅列（キーワードスタッフィング）は避ける
- サイト名やブランド名をタイトル末尾に付けるかどうかは、運用ルールに合わせて柔軟に（必要ならコメントで提案）
- 元記事の内容と齟齬がないように、実際の本文をよく読んでから生成すること
